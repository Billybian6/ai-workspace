## Windows 中文编码规则

当前环境是 Windows，中文文件可能因为 PowerShell 默认编码导致乱码。

读取中文 txt/md/csv 文件时：
1. 优先使用 Python 读取：
   open(path, "r", encoding="utf-8-sig", errors="replace")
2. 如果使用 PowerShell，必须显式指定：
   Get-Content -Encoding UTF8 "文件路径"
3. 不要直接使用：
   Get-Content "文件路径"
4. 如果 UTF-8 读取仍然乱码，再尝试 GBK：
   open(path, "r", encoding="gbk", errors="replace")
5. 输出中文文件时统一使用 UTF-8 with BOM 或 UTF-8。

## 管理人产品文档工作流

我的管理人材料统一存放在：

C:\Users\bianzhengzhi\Desktop\管理人

当我说“为某个管理人生成产品文档”时，默认规则如下：

1. 管理人名称 = 我输入的名称，例如“进化论”“子午”。
2. 材料目录默认推断为：
   C:\Users\bianzhengzhi\Desktop\管理人\{管理人名称}
3. 输出目录默认推断为：
   C:\Users\bianzhengzhi\Desktop\管理人\{管理人名称}\{管理人名称}-产品文档
4. 输出文件默认推断为：
   C:\Users\bianzhengzhi\Desktop\管理人\{管理人名称}\{管理人名称}-产品文档\{管理人名称}-产品文档.md
5. 默认使用 prompt：
   C:\Users\bianzhengzhi\Desktop\AI工作台\prompts\产品文档梳理.md
6. 只生成一个 Markdown 文件，不要拆分多个文件。
7. 不要修改原始材料，只允许在输出目录中新建或更新产品文档。
8. 如果材料目录不存在，先提醒我，不要自行创建管理人材料目录。
9. 如果有多个相似文件夹名，先列出候选文件夹，让我确认。


工作目标：
1. 帮我从管理人材料中提炼产品策略、收益来源、风险来源、业绩特征、容量约束、流动性安排、费率要素、适配客户和尽调追问。
2. 生成的文档要适合我向领导汇报、内部评审、销售培训和后续尽调使用。
3. 所有结论必须基于材料，不能编造。材料没有的信息要写“待确认”。
4. 如果不同材料之间存在冲突，要单独列出“材料冲突点”。
5. 输出要结论先行，语言正式、克制，不夸大收益，不承诺未来表现。
6. 未经明确允许，不要修改原始材料。只允许在指定输出文件夹中新建或更新文件。
7. 分析 Excel 时，要说明数据口径、时间区间、缺失值、异常值和可能的数据问题。
8. 涉及私募产品时，重点关注：收益来源、回撤、波动、容量、交易拥挤、风格漂移、流动性、合规风险、极端行情表现。
