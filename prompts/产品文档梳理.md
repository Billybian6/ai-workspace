# 任务：生成管理人产品文档

你现在要基于指定管理人文件夹中的材料，生成一份结构化产品文档。

## 输入
- 管理人名称：{{MANAGER_NAME}}
- 材料目录：{{MATERIAL_DIR}}
- 输出目录：{{OUTPUT_DIR}}

## 工作规则
1. 先只读材料，不要修改原始文件。
2. 优先读取已经转换好的 txt、md、csv 文件；如果没有转换文件，再读取原始文件。
3. 每个重要结论后面尽量标注来源文件名、表名、sheet 名或材料位置。
4. 不确定的信息不要猜，统一写入“待确认事项”。
5. 如果材料之间有冲突，单独列为“材料冲突点”。
6. 只允许在 {{OUTPUT_DIR}} 下生成文件，不要改动其他目录。

## 输出文件
请只生成一个文件：
{{MANAGER_NAME}}-产品文档.md

所有内容都写入这一个 Markdown 文件，不要拆分成多个文件。文件内部按章节组织。

### 01_管理人产品总览.md
包括：
- 管理人基本情况
- 投研团队与核心人员
- 主要策略线
- 旗下产品谱系
- 产品之间的关系
- 我对这家管理人的一句话理解

### 02_旗下产品策略拆解.md
对每只产品分别分析：
- 产品名称
- 策略类型
- 策略目标
- 收益来源
- 交易频率与持仓周期
- 标的范围
- 风控方式
- 容量判断
- 适合市场环境
- 不适合市场环境
- 与同类产品的差异

### 03_核心产品分析.md
挑出最值得重点关注的产品，详细分析：
- 为什么值得重点关注
- 业绩特征
- 回撤和波动特征
- 收益稳定性
- 极端行情表现
- 是否适合代销
- 适合什么客户
- 引入前还需要确认什么

### 04_风险点与尽调追问.md
分模块列出：
- 策略风险
- 业绩风险
- 容量风险
- 流动性风险
- 交易与风控风险
- 团队稳定性风险
- 合规与运营风险
- 需要追问管理人的问题

每个问题后面给出追问逻辑，说明为什么要问。

### 05_销售话术与适配场景.md
包括：
- 一句话介绍
- 三句话介绍
- 给销售的简短版本
- 给领导汇报的正式版本
- 适合客户画像
- 不适合客户画像
- 不能对客户说的话

## 输出风格
- 结论先行
- 正式但不要空泛
- 尽量用表格
- 不要写成宣传稿
- 不要夸大收益
- 不要使用“稳健收益”“确定性收益”“保本”等不合规表达
